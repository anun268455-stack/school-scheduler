4.2 ห้องสอน (Rooms)

⚙ จัดการ
→
🚪 ห้องสอน
ฟิลด์ความหมายตัวอย่าง
ชื่อห้อง *ชื่อห้องห้อง 101, ห้องวิทย์ 1
ประเภทลักษณะของห้องทั่วไป / พิเศษ / กลางแจ้ง / ห้องเวียน
อาคารอาคารที่ตั้งอาคาร 1 (หลัก)

ℹ️
ประเภทห้อง:
• ทั่วไป — ห้องเรียนปกติ
• พิเศษ — ห้องแล็บ, ห้องคอม ใช้ได้เฉพาะวิชาที่กำหนด
• กลางแจ้ง — สนามกีฬา, ลาน ใช้กับพลศึกษา
• ห้องเวียน — ห้องที่นักเรียนเดินมาหา ไม่ใช่ห้องประจำ

4.3 ครูผู้สอน (Teachers)

⚙ จัดการ
→
👨‍🏫 ครู
ฟิลด์ความหมายค่าแนะนำ
ชื่อครู *ชื่อ-นามสกุลครูสมชาย ใจดี
กลุ่มสาระฯหมวดที่สังกัดเลือกจากรายการ
คะแนนกลางแจ้ง (0-10)ความเหมาะสมสอนกลางแจ้งพลศึกษา = 10, อื่นๆ = 3-5
สอนสูงสุด/วันคาบสูงสุดต่อวัน5-6 คาบ
กลางแจ้งสูงสุด/สัปดาห์จำกัดคาบกลางแจ้ง0-2 คาบ
💡
ค่าเหล่านี้มีผลต่อ Solver — ครูที่ตั้ง "สอนสูงสุด/วัน = 4" จะไม่ถูกจัดมากกว่า 4 คาบต่อวัน

4.4 วิชาเรียน (Subjects)

⚙ จัดการ
→
📚 วิชา
ฟิลด์ความหมายตัวอย่าง
รหัสวิชา *รหัสสั้นๆMATH101, SCI201
ชื่อวิชา *ชื่อเต็มคณิตศาสตร์, ฟิสิกส์
ประเภททั่วไป หรือ คู่ขนานดูด้านล่าง
จำนวนคาบ1 หรือ 2 คาบต่อครั้งพลศึกษา = 2 คาบ
น้ำหนักเบา/หนัก (แสดงสีในตาราง)คณิต/วิทย์ = หนัก
กลุ่มสาระฯหมวดที่สังกัดเลือกจากรายการ
วิชากิจกรรมtick ถ้าเป็นชุมนุม/ลูกเสือชุมนุม, ลูกเสือ
ℹ️
ประเภทวิชา:
• ทั่วไป (common) — สอนแต่ละห้องแยกกัน ต่างเวลากันก็ได้
• คู่ขนาน (parallel) — หลายห้องเรียนพร้อมกัน เช่น พลศึกษา ม.1 ทุกห้อง

⏰ คาบ & เวลา

กำหนดว่าแต่ละคาบเริ่มและสิ้นสุดเมื่อใด และเป็นคาบประเภทใด

ประเภทคาบความหมายผลต่อ Solver
classคาบเรียนปกติSolver จะจัดวิชาลงคาบนี้
breakคาบพักไม่จัดวิชา แสดงเป็นสีเทาในตาราง
lunchคาบกินข้าวไม่จัดวิชา

homeroomโฮมรูมตอนเย็นไม่จัดวิชา
⚠️
ระบบมาพร้อมคาบตัวอย่างแล้ว ถ้าตรงกับตารางโรงเรียน ไม่ต้องแก้ไข
ถ้าเวลาต่างออกไป ให้กด "แก้ไข" ด้านขวาของแต่ละคาบ

การเพิ่มคาบใหม่

1

2
ชื่อคาบเช่น "คาบ 1", "พักกลางวัน"
3
เลือกประเภทclass / break / lunch / assembly / homeroom
4
กรอกเวลาเริ่ม-สิ้นสุดรูปแบบ HH:MM เช่น 08:30
5
ใช้กับall = ทุกระดับ, lower = ม.1-3, upper = ม.4-6

วิธีรัน Solver

1
กด "⚡ Solver" ใน Toolbarหน้าต่าง Solver จะเปิดขึ้น
2

3
กด "▶ รัน Solver"ระบบจะทำงาน 10-120 วินาที
4
ดูผลลัพธ์จำนวนคาบที่สร้าง, เวลาที่ใช้, และการละเมิด (ถ้ามี)

ถ้ามีคาบที่ต้องการกำหนดเวลาตายตัว (เช่น คาบเคารพธง, ชุมนุม, คาบพิเศษ) ให้ ล็อคก่อนรัน Solver
Solver จะไม่ย้ายคาบที่ล็อคไว้
1
กดปุ่ม "🔓 Lock" ใน Toolbarปุ่มจะเปลี่ยนเป็น "🔒 Lock ON" สีเหลือง
2
คลิกที่คาบที่ต้องการล็อคคาบที่ล็อคจะแสดงไอคอน 🔒
3
กด Lock อีกครั้งเพื่อออกจาก Mode
4
รัน Solverคาบที่ล็อคจะคงอยู่เดิม

การใช้งานตารางเรียน

8.1 การดูตาราง (3 มุมมอง)

มุมมองปุ่มแสดงอะไรเหมาะกับ
ตารางห้องเรียน👥 ห้องตารางเรียนของห้องที่เลือกนักเรียน, ผู้บริหาร
ตารางครู👨‍🏫 ครูตารางสอนของครูที่เลือกครู, ฝ่ายบุคคล
ตารางห้องสอน🚪 ห้องสอนการใช้ห้องแต่ละห้องฝ่ายอาคาร, ผู้บริหาร
1
กดปุ่มมุมมองที่ต้องการเช่น กด "👨‍🏫 ครู"
2
เลือกชื่อครู/ห้อง จาก Dropdownตารางจะแสดงทันที

การอ่านสีและสัญลักษณ์ในตาราง

สี / สัญลักษณ์ความหมาย
สีเหลืองอ่อน ☀วิชากลางแจ้ง (พลศึกษา, สนาม)
สีน้ำเงินอ่อนวิชาหนัก (คณิต, วิทย์)
สีเขียวอ่อนคาบคู่ขนาน (หลายห้องพร้อมกัน)
🔒คาบที่ถูกล็อค
2×คาบคู่ (Double Period)
เทา/ลายเส้นคาบพัก/กินข้าว (ไม่จัดวิชา)

8.2 การลากวางคาบเรียน (Drag & Drop)

1
คลิกค้างที่คาบที่ต้องการย้ายเคอร์เซอร์จะเปลี่ยนเป็นรูปมือ
2
ลากไปยังช่องที่ต้องการจะเห็นสีแสดงผลกระทบ
3
วางเมาส์คาบจะย้ายทันที หรือเปิด Modal ถ้ามีความขัดแย้ง

ความหมายของสีแสดงผลกระทบ

สีความหมายทำได้ไหม
🟢 เขียวปลอดภัย ไม่มีผลกระทบหรือน้อยมากวางได้เลย
🟡 เหลืองต้องสลับ 2-4 คาบ มีผลกระทบปานกลางวางได้แต่ควรระวัง
🔴 แดงไม่สามารถวางได้ (ห้ามตามกฎหลัก)วางไม่ได้
⚫ เทาคาบปัจจุบัน (เดิม)—

8.3 หน้าต่างแก้ไขความขัดแย้ง (Conflict Resolution Modal)

⚡
จะปรากฏเมื่อลากคาบไปวางในช่องที่มีครู/ห้องซ้ำกัน
ตัวเลือกความหมายใช้เมื่อไหร่
🔄 Path A — สลับคาบ (Swap)ย้ายคาบที่ขัดแย้งมาอยู่ตำแหน่งเดิมของคาบที่กำลังลากต้องการสลับสองคาบ
⚠️ Path B — บังคับวาง (Force)วางทับ คาบเดิมจะถูกลบออกจากตารางแน่ใจว่าต้องการลบคาบเดิม
ยกเลิกยกเลิกการย้ายไม่ต้องการทำอะไร

8.4 การล็อคและปลดล็อคคาบ

🔒 ล็อคทีละคาบ

  1. กด "🔓 Lock" ใน Toolbar → ปุ่มเปลี่ยนเป็นสีเหลือง
  2. คลิกคาบที่ต้องการล็อค
  3. กด "🔒 Lock ON" อีกครั้งเพื่อออก

🔒🔒 ล็อคทั้งหมดพร้อมกัน

  1. กด "🔒 ทั้งหมด" ใน Toolbar
  2. คาบทุกคาบจะถูกล็อคทันที
  3. กด "🔓 ปลด" เพื่อปลดล็อคทั้งหมด

คำถามที่พบบ่อย (FAQ)

❓ Solver รันแล้วได้ 0 คาบ ทำอย่างไร?

  • ตรวจสอบว่า Backend รันอยู่ (เปิด PowerShell แล้วรัน python mock_api.py)
  • ตรวจสอบว่ามี Requirements อย่างน้อย 1 รายการ
  • ตรวจสอบว่ามีคาบ type=class อย่างน้อย 1 คาบใน Periods

❓ ข้อมูลหายหมดเมื่อเปิดโปรแกรมใหม่?

  • ระบบปัจจุบันใช้ RAM จัดเก็บ ข้อมูลจะหายเมื่อ Backend หยุดทำงาน
  • ถ้าต้องการเก็บข้อมูลถาวร ต้องเพิ่มฐานข้อมูล (SQLite/PostgreSQL) — แจ้งทีมพัฒนา

❓ ลากวางคาบไม่ได้ ทำอย่างไร?

  • ตรวจสอบว่าคาบนั้นไม่ได้ถูก "ล็อค" (มีไอคอน 🔒)
  • ช่องปลายทางที่แสดงสีแดง 🔴 = วางไม่ได้ ต้องเลือกช่องอื่น
  • ถ้าต้องการย้ายคาบที่ล็อค ต้องปลดล็อคก่อน

❓ ตารางพิมพ์ออกมาหัวข้อว่าง ทำอย่างไร?

  • ไปที่ ⚙ จัดการ → 🏫 ตั้งค่าโรงเรียน
  • กรอกชื่อโรงเรียน ภาคเรียน ปีการศึกษา
  • กด "บันทึกการตั้งค่า" แล้วลองพิมพ์ใหม่

❓ วิชาคู่ขนานจัดตารางไม่ตรงกัน ทำอย่างไร?

  • ตรวจสอบว่า Requirements ของทุกห้องที่เรียนพร้อมกัน มี Parallel Key เดียวกัน
  • ตัวอย่าง: พลศึกษา ม.1/1, ม.1/2, ม.1/3 ต้องมี Parallel Key = PE-M1-001 ทุกรายการ

❓ เพิ่มข้อมูลแล้วไม่ปรากฏในตาราง ทำอย่างไร?

  • ต้องรัน Solver ใหม่หลังเพิ่ม Requirements
  • Solver ไม่ได้รันอัตโนมัติ ต้องกด ⚡ Solver ทุกครั้งที่แก้ไข Requirements

❓ เว็บออนไลน์เปิดแล้วช้ามาก?

  • Render.com แบบฟรีจะ sleep เมื่อไม่มีคนใช้ รอ 30-60 วินาที ระบบจะ wake up เอง
  • ครั้งแรกของวันอาจช้า ครั้งต่อไปเร็วขึ้น

❓ ต้องการ Deploy ขึ้นออนไลน์ใหม่หลังแก้โค้ด ทำอย่างไร?

git add .
git commit -m "อธิบายการแก้ไข"
git push

GitHub Actions จะ Build และ Deploy ให้อัตโนมัติใน 3-5 นาที
